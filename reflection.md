I started from the Ralph Wiggum loop shown in class and rewrote it into my own script [sidd.sh](sidd.sh). I fist started with just the build loop but then I thought it would be nice to have plan also happening within the loop. So the final script has two modes: a plan loop and a build loop. In these loops every iteration is a fresh agent with no memory and the build loop ends in a critic pass that checks the code against SPEC.md by actually executing things. My theory was that if I wrote a hard machine-checkable spec interactively, I could let the plan loop slice it into file-disjoint waves (I wanted to include parallel execution also) and the build loop grind through them without me. That mostly worked and the critic even formally approved the base build. However, the site it approved looked bad, overlapping layouts and a color scheme that ignored the spec. Every criterion was green because I had only encoded what I knew how to check. So I went in manually, diagnosed from screenshots, and updated the fix wave straight into PLAN.md myself instead of rerunning the plan loop, which for a short list of concrete defects would have just over-engineered things, then let the build loop run again to refine. I repeated the whole cycle for the game extension. However, for the game's refine step I updated SPEC.md manually but used the plan loop again. Everything else was the same. What stuck with me is that a spec really is a contract, and an autonomous loop satisfies exactly the contract and nothing more, so the quality of the result is bounded by what you can express as a check, and the human's job shifts from writing code to noticing what the checks cannot see and feeding that back in.

My honest reflection is that this is definitely not enough as-is. I had to manually do quite a bit of work as a part of refine and even then the website we have live can still use a lot of refinement. I think giving agents the power to verify in almost an exhaustive manner would really change things here. The process that I went through assumed that we knew all requirements for build at plan stage and all requirements for plan and spec stage. I think this is not how things normally take shape. While this was a fun experience, I think ralph loop is probably better here. However, thinking more deeply I feel some things are certain to me after trying this. 1) I feel having as much context as possible in the beginning is really helpful and even critical. We need to find ways to add that context. That could be by answering a lot of questions or could be by really doing a lot of research, but I just feel that we should have the vision very clear in the beginning and we should be as specific as possible right in the beginning. I think that would really help. 2) I also feel we should think about loops in terms of ending with verification, and that verification layer seems to be becoming a lot more important than ever before.  Since the human is not in the loop, the AI agent itself has to do the verification, and in that case, either we should be vague but then give an exhaustive scope or we need to be very specific about the type of testing that we actually want and actually want the agen to do. In my case, I feel I did not work on the verification part enough because I myself like to open pages and then tell AI, e(ven sometimes clicking screenshots and giving it to AI to do stuff) and that simply wasn't a part of my loops, and that's why I my front end part was not very well done. So I think the onus is then on the user running the loop to make sure that this verification is really good.

Overall, I really enjoyed this exercise and while I have used in-built loops Claude Code and Codez, this is the first time I actually wrote a agent loop and I think this was really good experience because I feel I have a much better understanding of the paradigm now. I'm really excited to see where this paradigm takes the industry and it seems very promising. I do think that this also creates a mental model shift that we all need to quickly adapt to because this seems like the more efficient way to eventually use AI agents (even though it wastes a lot of tokens right now).
